Project Overview




The app was created to address the need for a user-friendly platform that allows movie enthusiasts to explore a comprehensive catalog of films, register an account, and personalize their experience by managing a list of favorite movies. The goal was to build an engaging Angular-based front end for the existing myFlix API, adhering to RESTful design principles.

The myFlix-Angular-client project was developed to provide movie enthusiasts with an intuitive platform to explore films, learn about genres and directors, and manage their favorite movies. Built using Angular, the app integrates seamlessly with a RESTful API to deliver a dynamic, responsive user experience.

Project Planning




The project began with outlining user journeys, focusing on essential features like user registration, login, movie browsing, and personalized functionalities. My primary goal was to create a responsive and visually engaging interface that worked across all devices. However, I faced an initial hurdle in selecting the right framework for the design. After researching options, I chose Angular Material for its versatile pre-built components and responsive grid system, which greatly simplified the design process.

Backend Integration






With the front-end design underway, I moved on to integrating the app with my existing RESTful API. Using Angular’s HttpClient module, I centralized all API calls into a service file. One of the challenges I faced was managing asynchronous API calls without causing delays or errors in the UI. To address this, I implemented RxJS observables and error-handling mechanisms using Angular’s catchError operator. This ensured a smooth, error-free experience for users.

Front-End Development









The app’s interface was developed using Angular Material components and a flexible routing system. One of the more complex tasks was ensuring that user interactions, such as adding or removing favorite movies, dynamically updated the UI in real-time. By leveraging Angular’s two-way data binding, I was able to sync user actions with the interface seamlessly, creating a more interactive and user-friendly experience.

Documentation







To ensure the app was well-documented for future developers, I used TypeDoc. Writing concise yet comprehensive documentation was challenging at first, but by adding structured comments throughout the codebase, I created clean, organized, and accessible documentation for the project.

Summary




Final Solution

The app uses Angular with Angular Material for a modern, responsive UI. MongoDB stores user data and movie details, while Mongoose facilitates seamless data handling. Firebase handles secure authentication, and the app is optimized for both desktop and mobile devices.


Challanges

Integrating MongoDB with the backend was the most challenging aspect, particularly around structuring the database and managing dynamic features like user favorites. Firebase authentication also required precise syncing between the frontend and backend, involving careful API configuration.


Future Improvements

While the project achieved its goals, future iterations could add features like movie search filters, recommendations, and reviews. Testing coverage, especially for API calls and authentication, can also be improved. This project was a valuable learning experience in integrating Angular, MongoDB, and Firebase, and I’m proud of the end result.
